Introduction to Attendance Manager

I started off this project on the Android Application Development Platform. I had been working on small Android Applications since a long time and wanted to try something new. I started off with a simple list interface where each row would toggle its state on a click. I tried to build a decent User Experience until I decided to let our HOD in on my secret project. Prof. Joshi were totally aboard with the idea of an Attendance Manager for our department. Prof. Joshi, being very supportive of the idea, asked me to talk to Prof. Mayur Parulekar to kickstart the project ASAP. I discussed the basic functionality with Prof. Parulekar and started the project right away.

Enough with the history, lets begin with the technical details!!

As mentioned earlier, I started off the project on the Android App Dev Platform. Based on my conversations with Prof. Joshi and Prof. Parulekar, they wanted me to develop a standalone application i.e. no Client-Server Architecture and the attendance records would be stored on each individual teacher's phones (in the Internal Storage).

Since it was a standalone model, I decided not to use a DatabaseEngine. Instead, I would store all the data on the users' phones as files in a custom folder that I'd create the first time the app is run.

I decided to use the JSON (JavaScript Object Notation) format for data storage in each of those files. The primary reason being the popularity and acceptance of JSON across the entire Software Dev Industry.

Transition to Web
The Web Dev followed a similar data storage approach as described above except it was a Client Server Model now; and most importantly it was WEB DEV now!
Since it was WebDev now, I had to code everything from scratch, only the logic would be same.
During the development phase, my local machine, i.e. my PC would be my Server (obviously). When we started to test the functionality in college with professors, the Server was run on an Ubuntu Linux VM rented at DigitalOcean.
For the development, I decided to use the MEAN Stack -
MongoDB - DatabaseEngine,
ExpressJS - Web Application Framework,
Angular - Web Application Framework (Frontend - Client-side programming),
NodeJS - JS run-time environment (Backend - Server-side programming)

Refer to the official documentation to get started.
